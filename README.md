# Fine-tuning Mistral-7B на английских инструкциях

Дообучаю `mistralai/Mistral-7B-Instruct-v0.3` на датасете
`argilla/ultrafeedback-binarized-preferences-cleaned` — 60 917 инструкций на
английском языке с оценками качества от GPT-4 (шкала 1–5).

**Цель:** улучшить способность модели следовать инструкциям, обучившись на
лучших ответах из UltraFeedback.

**GPU:** RTX 5060 Laptop GPU, 8GB VRAM, архитектура Blackwell (sm_120), CUDA.

---

## Датасет и фильтрация

Взял UltraFeedback — это не какой-то один датасет, а сборка из 8 источников:
Evol-Instruct, ShareGPT, UltraChat, FLAN, FalseQA, TruthfulQA и других. Для
каждого примера есть ответ от модели и его оценка GPT-4 (chosen-rating).

Фильтровал так:
- Оставил только примеры с `chosen-rating >= 4.5` — то есть только те, где
  GPT-4 оценил ответ как отличный. Из 60 917 строк прошло ~50 000.
- Убрал дубликаты по полю `prompt` — одинаковые инструкции не нужны.
- Взял первые 1000 уникальных на train (все из Evol-Instruct — там
  инструкции самые разнообразные по тематике, эволюционирующие в сложности).
- Ещё 100 примеров (тоже с rating >= 4.5) отложил на test — следующая
  тысяча после train.

Так я получил чистый набор из 1000 качественных инструкций для обучения и
100 для оценки. Никакой случайной выборки — просто sequential cut, чтобы
можно было воспроизвести.

## Обучение: QLoRA под RTX 5060

Mistral-7B весит ~14 GB в fp16. У меня 8 GB. Значит, без квантизации никак.

### 4-bit NF4 — почему именно это

QLoRA предложила формат NF4 (NormalFloat4) — он адаптирован под нормальное
распределение весов нейросетей. Вместо равномерного шага квантования (как в
обычном int4) он распределяет уровни так, чтобы плотнее покрывать область
около нуля, где находится большинство весов. На практике это даёт качество
близкое к fp16 при 4-кратном сжатии. Модель в 4-bit весит ~3.5 GB.

### LoRA r=16, alpha=32

LoRA — это low-rank адаптеры: вместо дообучения всех 7 млрд параметров мы
учим маленькие матрицы ранга r, которые потом добавляем к attention-слоям.

- `r=16` — золотая середина. r=8 было бы слишком мало для instruction
  tuning (не хватит ёмкости), r=32 не влезет в VRAM (больше параметров →
  больше градиентов).
- `alpha=32` — стандартное правило `alpha = r * 2`. Это масштабирует
  адаптер так, чтобы update был соизмерим с оригинальными весами.

Target modules — `q_proj` и `v_proj` (query и value в attention). Это
стандартный выбор для instruction tuning: attention — ключевой механизм для
понимания инструкций.

### Оптимизация под 8GB VRAM

| Проблема | Решение |
|----------|---------|
| Модель не влезает целиком | 4-bit NF4 (~3.5 GB вместо 14) |
| Batch=4 даёт OOM | batch_size=2 + gradient_accumulation=8 → effective batch 16 |
| Не хватает памяти на активации | gradient_checkpointing (пересчёт на backward вместо хранения) |
| Адаптер жрёт VRAM | target_modules только q_proj, v_proj (не все) |
| Long sequences жрут память | max_seq_length=512 (средняя длина инструкции 77 слов) |
| Оптимизатор ест память | paged_adamw_8bit (страницы на CPU при нехватке GPU) |
| bf16 глючит на Blackwell | оставил bf16 — RTX 5060 поддерживает нативно |

В итоге: обучение на 1000 примерах заняло ~49 минут (189 шагов, 3 эпохи).
Loss упал с 1.25 до 0.84.

## Инструменты

**Hugging Face:**
- `datasets` — загрузил UltraFeedback, отфильтровал, посмотрел статистику.
  Не пришлось ничего качать вручную, всё через API.
- `transformers` — модель, токенизатор, BitsAndBytesConfig для 4-bit.
- `trl` (SFTTrainer) — обучение на инструкциях. TRL 1.6.0, новый API с
  SFTConfig.
- `peft` — LoRA адаптер. Заморозил оригинальные веса, обучаю только ~6.8M
  параметров (0.09% от всех).
- `bitsandbytes` — 4-bit квантование на GPU.

**Weights & Biases** — залогировал loss, grad_norm, learning_rate,
accuracy. Удобно смотреть кривые в реальном времени и сравнивать run'ы:

https://wandb.ai/alen-makanov23-apps/ultrafeedback-sft

## Оценка

Сравнивал базовую Mistral и дообученную на 10 тестовых примерах (те же
UltraFeedback с rating >= 4.5, но модель их не видела).

**Процесс:**
1. Гружу базовую модель, генерирую ответы на 10 инструкций (temperature=0.3,
   max_new_tokens=100).
2. Выгружаю, чищу кэш, гружу finetuned модель (base + LoRA adapter).
3. Генерирую ответы на те же инструкции.
4. Сохраняю оба набора в JSON.

### LLM-as-a-Judge (Gemini Pro)

Отправил оба файла (base.json и finetuned.json) в Gemini Pro с промптом:
сравнить ответы A vs B вслепую по критериям helpfulness, correctness,
completeness и стиля. Модель-судья не знала, какой файл какой.

Результаты на 10 инструкциях:

| Исход | Доля |
|-------|------|
| Finetuned лучше | 50% (5/10) |
| Tie (одинаково) | 30% (3/10) |
| Base лучше | 20% (2/10) |

**Вывод:** в 80% случаев (победы + ничьи) дообученная модель не хуже
базовой. В половине случаев — заметно лучше. Базовая выиграла только там,
где нужно было строго следовать жёсткому формату ответа (видимо,
fine-tuning немного «разболтал» стиль).

### Дополнительно: ROUGE-L

Для объективности посчитал ROUGE-L на 20 примерах относительно референсов
из UltraFeedback (оригинальные ответы от GPT-4):

```
                ROUGE-L
Base            0.1767
Finetuned       0.2038
Improvement     +15.4%
```

ROUGE-L тоже показывает улучшение, но абсолютные значения низкие, потому
что метрика измеряет лексическое совпадение, а не качество ответа.

### Выводы

**Что хорошего:**
- LLM-as-a-judge подтвердил: дообученная модель объективно лучше в 50%
  случаев, baseline — только в 20%.
- Loss упал с 1.25 до 0.84, ROUGE-L вырос на 15% — все метрики смотрят в
  одну сторону.
- LoRA сработала: 6.8 млн параметров (0.09% от всех) хватило, чтобы
  сдвинуть поведение модели.
- QLoRA на 8GB VRAM — рабочий вариант. 49 минут на 1000 примеров — норм.

## Структура репозитория

```
├── data/
│   ├── train.jsonl          # 1000 инструкций (обучение)
│   └── test.jsonl           # 100 инструкций (оценка)
├── scripts/
│   ├── 01_prepare_data.py   # Загрузка, фильтрация, дедупликация
│   ├── 02_train.py          # QLoRA обучение (SFTTrainer)
│   └── 03_evaluate.py       # ROUGE-L: base vs finetuned
├── model/
│   └── adapter/             # LoRA веса + токенизатор
├── eval/
│   ├── base.json            # Ответы базовой модели
│   ├── finetuned.json       # Ответы дообученной модели
│   └── summary.json         # ROUGE-L результаты
├── requirements.txt
└── README.md
```

## Как запустить

```bash
pip install -r requirements.txt

# 1. Подготовить данные
python scripts/01_prepare_data.py

# 2. Обучить
python scripts/02_train.py

# 3. Оценить
python scripts/03_evaluate.py
```
